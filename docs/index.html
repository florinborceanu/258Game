1. Despre:

Raportul de fata curprinde detaliile care au vizat progresul proiectului 'AnCoWe', denumit ulterior si 'ScharaK Online' propus in cadrul laboratorului de Tehnologii Web. Sunt surprinse in prezentul raport detalii legate de obiectivele proiectului si etapele realizarii acestor obictive, cat si date legate de folosirea sistemului de administrare.

2. Introducere:

ScharaK Online sau AnCoWe este o aplicatie web ce contine un joc de tip Clicker RPG ce recurge la API-ul oferit de MARVEL pentru a primi imaginile caracterelor inamice, numele si o scurta descriere a fiecaruia. Aplicatia noastra se adreseaza jucatorilor „casual”, cu varste cuprinse intre 12 si 18 ani. Odata conectat pe site-ul aplicatiei, userul are o varietate de alegeri. De la crearea contului la lupta propriuzisa, toata interactiunea acestuia este ghidata si sugestiva.

3. Obiective:

  • Sa se realizeze un joc Web de tip RPG (Role-Playing Game) care va recurge la date publice privitoare la anime (e.g., MyAnimeList) si la cele despre eroi de benzi desenate (precum Marvel API) pentru a reconstitui, la alegere, scenariul unui joc disponibil pe platforme clasice.

  • Realizarea unui clasament, decizia luata a fost de a arata in acesta doar primii 6 jucatori.
  • Implementarea unui API de tip REST, descris via OpenAPI Specification utilizat in aplicatie pentru a primi date despre fiecare jucator si ofera date legate de acestia - utilizate in clasamentul din Rankings.

4. Etape de dezvoltare:

  • Definitivarea unor elemente legate de interfata - etapa ce a continuat si pe parcurs
  • Modificarea bazei de date pentru a indeplinii noile necesitati
  • Implementarea MVC web a site-ului in PHP
  • Dezvoltarea mecanicii jocului in JavaScript
  • Integrarea API-ului oferit de Marvel in joc
  • Crearea de animatii de atac
  • Implementarea API-ului
  • Integrarea API-ului creat in joc
  • Integrarea API-ului creat pentru generarea tabelei Rankings
  • Implementarea Login si Register
  • Implementarea Change Email, Username, Password

5. Sistemul de management al codului-sursă:

Pentru a administra codul-sursă am ales sistemul celor de la Github. Am folosit Github Desktop pentru o utilizare usoara. Modificarile facute in directorul proiectului sunt vizibile si pot si pot fi adaugate proiectului prin succesiunea de comenzi 'commit' si 'push origin' care pot fi realizate prin simpla apasare a unor butoane din aplicatia Github Desktop. Pentru a prelua local schimbarile facute de ceilalti contribuitori se aplica, la fel de simplu, comenzile 'fetch origin' si apoi 'pull origin'. Commit-urile au fost adaugate ('merge') branch-ului master.

6.Concluzie

Proiectul a fost finalizat cu succes in ciuda dificultatilor intampinate si am reusit sa atingem toate obiectivele propuse.
